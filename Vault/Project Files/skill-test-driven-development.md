# skill-test-driven-development

## מה הסקיל עושה
מחייב כתיבת בדיקות לפני הקוד (TDD) בעת יישום פיצ'ר או תיקון באג. נחשב לסקיל "rigid" — יש לפעול בדיוק לפי השלבים.

## מתי להפעיל
בכל יישום של פיצ'ר/באגפיקס, לפני כתיבת קוד יישום.

## קבצים בתת-התיקייה
- **`SKILL.md`** — המסמך הראשי.
- **`testing-anti-patterns.md`** — אנטי-פטרנים נפוצים בכתיבת בדיקות שצריך להימנע מהם.

## למי הוא משויך
- **תפקיד:** משמעת איכות בקוד.
- **קוראים:** Claude Code בעת יישום.
