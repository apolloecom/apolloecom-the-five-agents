# skill-obsidian-vault-workflow

## מה הסקיל עושה
**הסקיל המרכזי של הפרויקט.** מחייב פרוטוקול קריאה/כתיבה ב-Vault: בתחילת כל משימה — איתור קובץ הנושא וקריאת Overview + Session Log + Meeting Notes / Briefs / Brand Guidelines רלוונטיים. בסוף כל משימה — append של Session Log entry בתאריך ועדכון Overview אם השתנה תחום.

## מתי להפעיל
**בכל משימה** — coding, content, ארכיטקטורה, design, פיצ'ר, באגפיקס, review. מדלגים רק על שאלות read-only טהורות.

## קבצים בתת-התיקייה
- **`SKILL.md`** — המסמך הראשי (קובץ יחיד).

## למי הוא משויך
- **תפקיד:** זיכרון ארוך-טווח של הפרויקט.
- **קוראים:** Claude Code בכל משימה.

- `Vault/Publishing Log/` — לוג פרסומים ו-postmortems.
- `Vault/Brand Guidelines/` — קווים מנחים מותגיים.
